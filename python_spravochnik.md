## Урок 1 — Основы

### print — вывод на экран

Показывает текст или значение в терминале. Команды выполняются **сверху вниз**, по очереди.

```python
print("Привет, мир!")
```

> 💡 *(сюда впишу свой комментарий своими словами)*

---

### Переменные

Контейнер с именем, в который кладём значение. Значение можно **менять** в любой момент — поэтому и "пере-менные".

```python
name = "Sergio"   # положить значение в переменную (присвоить)
name = "Alex"     # перезаписать — теперь там "Alex"
age = 30
age = age + 5     # взять старое значение, прибавить 5, записать обратно → 35
```

**Важно:** `print("name")` и `print(name)` — это разное!
- с кавычками → выведет буквально текст `name`
- без кавычек → выведет **значение** переменной (`Sergio`)

---

### Типы данных

Python определяет тип **сам** (динамическая типизация). Проверить тип: `type(переменная)`.

| Тип | Что это | Пример |
|-----|---------|--------|
| `str` | текст (строка) | `"Sergio"` |
| `int` | целое число | `30` |
| `float` | дробное число | `1.75` |
| `bool` | правда / ложь | `True` / `False` |

```python
print(type(age))   # <class 'int'>
```

> Чтобы число стало текстом — берём его в кавычки: `"30"` это уже `str`, а не `int`.

---

### Арифметика

| Оператор | Действие | Пример | Результат |
|----------|----------|--------|-----------|
| `+` | сложение | `30 + 5` | `35` |
| `-` | вычитание | `30 - 5` | `25` |
| `*` | умножение | `30 * 5` | `150` |
| `/` | деление | `30 / 5` | `6.0` ⚠️ |
| `//` | целочисленное деление | `30 // 5` | `6` |
| `**` | степень | `30 ** 2` | `900` |
| `%` | остаток от деления | `10 % 3` | `1` |

> ⚠️ Обычное деление `/` **всегда** даёт `float` (с точкой), даже если делится ровно. Нужно целое — используй `//`.

---

### f-строки

Способ вставить переменные прямо в текст. Буква `f` перед кавычками говорит: "внутри `{}` будут переменные — подставь их значения".

```python
print(f"Меня зовут {name}, мне {age} лет.")
# Внутри {} можно даже считать:
print(f"Прошло {age * 365} дней.")
```

> Без `f` Python выведет буквально `{name}`, а не значение.

---

### round — округление

Округляет число. Второй аргумент — сколько знаков после запятой.

```python
round(3.14159, 2)   # 3.14
round(3.14159)      # 3
```

> ❗ **Главное правило:** округляй только **финальный** результат, который показываешь.
> Промежуточные вычисления держи с полной точностью — иначе накопится ошибка.

## Урок 2 — Условия

### if / elif / else

Заставляет программу **принимать решения**. Читается как обычная человеческая логика.

```python
temp = 20

if temp > 25:          # если жарко
    print("Шорты")
elif temp < 10:        # иначе если холодно
    print("Куртка")
else:                  # во всех остальных случаях
    print("Футболка")
```

- `if` — первое условие (обязательно)
- `elif` — "иначе если", своё условие (можно несколько подряд)
- `else` — ловит "всё остальное", своего условия нет (необязателен)

После `if`, `elif`, `else` всегда ставится **двоеточие** `:`.

---

### Операторы сравнения

Любое сравнение Python сводит к `True` или `False`.

| Оператор | Значение |
|----------|----------|
| `>` | больше |
| `<` | меньше |
| `>=` | больше или равно |
| `<=` | меньше или равно |
| `==` | **равно** (два знака!) |
| `!=` | не равно |

> ⚠️ Самая частая ошибка новичков:
> - `=` это **присвоить** (`age = 30`)
> - `==` это **спросить** "равно ли?" (`age == 30`)

Сравнивать можно не только числа, но и текст:

```python
if color == "красный":
    print("Стой")
```

---

### Отступы

В Python отступ (сдвиг вправо) показывает "**это относится к** `if`". В других языках для этого скобки `{}`, в Python — отступы.

```python
if temp > 25:
    print("это внутри if")   # с отступом → выполнится при True
print("это снаружи")         # без отступа → выполнится всегда
```

> Если ошибиться с отступом → `IndentationError`.

---

### Порядок условий

Python проверяет условия **сверху вниз** и останавливается на **первом** `True`. Остальное не смотрит.

→ Поэтому условия выстраивают **от узкого к общему**.

```python
# ОПАСНО (порядок ломает логику):
if temp < 30:        # при temp=28 сработает ЭТО
    print("Футболка")
elif temp > 25:      # сюда уже не дойдёт, хотя 28 — это шорты
    print("Шорты")
```

## Урок 3 — Циклы

Цикл = **повторение** действия много раз. Два вида:
- **`for`** — когда знаю сколько раз (приседания: ровно 20)
- **`while`** — когда повторяю пока условие держится (посуда: пока раковина не пуста)

### for + range()

`for` проходит по набору и для каждого элемента выполняет тело.
`range()` генерирует последовательность чисел.

```python
for i in range(5):      # 0, 1, 2, 3, 4
    print(i)
```

- `i` — «текущее число», на каждом круге своё
- тело (с отступом) выполняется на каждом круге

**range с разными аргументами:**

```python
range(5)         # 0,1,2,3,4        — от 0 до 5 (5 не входит)
range(1, 11)     # 1,2,...,10        — от и до
range(0, 10, 2)  # 0,2,4,6,8         — третий аргумент = шаг
```

> ⚠️ **Правая граница НЕ включается.** `range(1, 11)` даёт до 10, не до 11.
> Это источник ошибки **off-by-one** (ошибка на единицу). Всегда проверяй границу.

### Переменная-копилка

Чтобы что-то накопить в цикле — заводим переменную ДО цикла, меняем ВНУТРИ.

```python
total = 0
for i in range(1, 11):
    total = total + i     # на каждом круге добавляем
print(total)              # 55  ← print СНАРУЖИ цикла
```

> 💡 `print` снаружи (без отступа) — покажет итог один раз. Внутри — печатал бы каждый круг.

### while

Повторяет тело **пока условие истинно**. Каждый круг проверяет заново.

```python
number = 1
while number <= 5:
    print(number)
    number = number + 1   # ← «толкатель», обязателен!
```

### Бесконечный цикл

Главная опасность `while`: если переменная в условии не меняется — цикл **никогда не закончится** (зависнет).

```python
number = 1
while number <= 5:
    print(number)
    # забыли number = number + 1  → печатает 1 бесконечно!
```

> 🛑 Если поймал бесконечный цикл — **Ctrl+C** в терминале прервёт его.
> Отступ решает: «толкатель» с отступом → внутри цикла (правильно), без отступа → снаружи (зависнет).

### and / or

Соединяют несколько условий в одно.

```python
if number % 3 == 0 and number % 5 == 0:   # оба должны быть истинны
    print("FizzBuzz")
```

- `and` — истинно когда **оба** условия истинны
- `or` — истинно когда **хотя бы одно** истинно

> 💡 Иногда `and` можно заменить математикой («делится на 3 и на 5» = «делится на 15»),
> но вариант с `and` **читается понятнее** — видно намерение. Короче ≠ всегда лучше.

## 🔧 Шпаргалка по Git

Git — снимки проекта локально. GitHub — облако, куда их выкладываешь.
**Снимок = коммит.**

**Базовый цикл работы:**

```bash
git status                          # что изменилось?
git add имя_файла                   # подготовить файл к снимку (staging)
git commit -m "описание"            # сделать снимок с подписью
git push                            # отправить в облако (GitHub)
```

**Разовая настройка нового репозитория:**

```bash
git clone https://github.com/.../repo.git   # скачать репозиторий на комп
cd repo                                       # зайти в папку
```

**Что значат состояния `git status`:**
- `Untracked` — новый файл, Git за ним ещё не следит → нужен `git add`
- `Changes to be committed` — подготовлен, ждёт коммита
- `ahead of origin by N commits` — есть N локальных коммитов, не отправленных → нужен `git push`

> 💡 **Git постоянно подсказывает следующий шаг прямо в выводе** — читай его, там написано что делать.
> 💡 Сообщение коммита пиши осмысленно («Добавил циклы»), а не «фыва». Через месяц спасибо себе скажешь.

## 📌 Привычки и принципы

Не синтаксис, а правила хорошего кода. Их я собираю отдельно.

- **DRY** (Don't Repeat Yourself) — значение храним в **одном** месте (в переменной), а не дублируем числом по коду. Поменял в одном месте — пересчиталось везде.
- **Округляй только финал** — промежуточные вычисления держи точными.
- **Плоское лучше вложенного** — если можно не вкладывать `if` в `if`, не вкладывай. Меньше путаницы с отступами, легче читать.
- **Код пишут для людей** — не только для компьютера. Через месяц читать его будешь ты сам.
- **Тестируй на границах** — ровно 18, ровно 90, ровно 0. Баги прячутся именно там.
- **Проверяй оба сценария** — если есть `if/else`, прогони и тот и другой путь.
- **Сверяй вывод с заданием** — «запустилось» ≠ «правильно». Программа может работать и врать.
- **Рефакторинг** — после рабочего кода почисти: понятные имена, убери лишние переменные.
- **Код читается чаще, чем пишется** — поэтому выбирай понятное, а не самое короткое.
